Deel 1 — REST API & Controllers

💡

Kernbegrip: REST API

Een REST API is de digitale ober tussen client (React) en server (database). Verzoeken verlopen via HTTP en data wordt uitgewisseld als JSON.

HTTP-methoden & Statuscodes

HTTP-methoden

  • GET — data ophalen
  • POST — nieuw item aanmaken
  • PUT — volledig updaten
  • PATCH — gedeeltelijk updaten
  • DELETE — verwijderen

Statuscodes

  • 200 OK — geslaagde GET/PUT
  • 201 Created — na POST
  • 204 No Content — na DELETE
  • 400 BadRequest — foute data
  • 401 Unauthorized — niet ingelogd
  • 403 Forbidden — geen rechten
  • 404 NotFound — bestaat niet

Parameter-attributen

In een controller gebruik je attributen om aan te geven waar de data vandaan komt:

  • [FromRoute] — verplichte data in de URL: /api/cities/5
  • [FromQuery] — optioneel filter na ?: /api/cities?name=Gent
  • [FromBody] — grote data in de body (POST/PUT)
⚠️

ControllerBase vs Controller

Gebruik altijd ControllerBase voor APIs. Een gewone Controller heeft extra HTML/View-functies die je bij een API niet nodig hebt — onnodige ballast.

🔒

DTO — Data Transfer Object

Stuur nooit je database-Entity rechtstreeks terug. Een DTO filtert welke velden de klant mag zien. Zo lek je geen gevoelige data (wachtwoorden) en blijft je API stabiel als de database verandert.

Deel 2 — Entity Framework Core

EF Core is een ORM (Object-Relational Mapper): het vertaalt je C#-klassen naar databasetabellen zodat je geen SQL hoeft te schrijven.

Relaties

  • One-to-Many: één stad heeft veel bezienswaardigheden — gebruik ICollection<T> aan de "één"-kant
  • Many-to-Many: gebruik een koppeltabel geconfigureerd via UsingEntity in OnModelCreating
  • Eager Loading met .Include() en .ThenInclude(): laad geneste data direct mee, anders zijn geneste lijstjes leeg in je JSON

💡

Deferred Execution

Een IQueryable-query wordt pas uitgevoerd op het allerlaatste moment — bij .ToListAsync(). Je kunt dus eerst al je filters stapelen en pas daarna naar de database gaan.

Deel 3 — Architectuur & Patronen

Layered Architecture — de 4 lagen

Het project is opgesplitst in lagen waarbij elke laag één taak heeft (Separation of Concerns):

Repository Pattern

Een repository verbergt alle database-code achter een interface. De controller praat altijd tegen de interface — nooit rechtstreeks tegen de implementatie. Dit maakt testen mogelijk via mocking.

Dependency Injection (DI)

DI = een klasse vraagt zijn benodigdheden op via de constructor in plaats van ze zelf te maken. Het principe erachter heet IoC (Inversion of Control).

⚠️

Waarom geen AddSingleton voor repositories?

De DbContext is niet thread-safe. Met AddScoped krijgt elke gebruiker zijn eigen verbinding die na het verzoek netjes wordt afgesloten.

Strategy Pattern & environment-registratie

In Development gebruik je een nep-mailservice (logt naar console), in Production een echte cloudservice. Je wisselt van implementatie door enkel de registratie in Program.cs aan te passen:

if (builder.Environment.IsDevelopment()) {
    builder.Services.AddSingleton<IMailService, LocalMailService>();
} else {
    builder.Services.AddSingleton<IMailService, CloudMailService>();
}

Async / Await

Database-acties duren lang. Met async/await kan de server ondertussen andere verzoeken afhandelen — dit maakt je API schaalbaar. Gebruik altijd Task<T> als returntype en await bij elke database-call.

Deel 4 — AutoMapper & Filtering

AutoMapper

AutoMapper kopieert automatisch velden van een Entity naar een DTO. Je maakt een Profile-klasse die de mapping beschrijft. Properties met dezelfde naam worden automatisch gemapt.

  • Enkel item: _mapper.Map<CityDto>(entity)
  • Lijst: _mapper.Map<IEnumerable<CityDto>>(list)
  • PUT (bestaand updaten): _mapper.Map(updateDto, existingEntity)
  • Andere namen: gebruik .ForMember(dest => ..., opt => opt.MapFrom(src => ...))

Filtering, Searching & Paging — de gouden volgorde

Deel 5 — Logging, Config & Testing

Logging met ILogger

Injecteer ILogger<T> in je controller en gebruik de juiste log level:

  • Trace → Debug → Information → Warning → Error → Critical

IConfiguration lees je instellingen uit appsettings.json via een key: config["MailSettings:MailToAddress"]

Unit Testing & Mocking

Het AAA-patroon structureert elke test:

  • Arrange — alles klaarzetten
  • Act — de methode uitvoeren
  • Assert — controleren of het klopt

Mocking (Moq): maak een nep-object van een interface zodat je in isolatie kunt testen zonder echte database: new Mock<ICityRepository>() → .Setup(...).Returns(...) → .Object

🔁

TDD — Test-Driven Development

Schrijf eerst de test (die faalt ✗), dan pas de code om de test te laten slagen (✓), dan refactor. Red → Green → Refactor.

Deel 6 — Authentication & Authorization

🪪 Authenticatie

"Wie ben je?" — Paspoortcontrole: je bewijst je identiteit via Auth0 en ontvangt een JWT-token.

🎫 Autorisatie

"Wat mag je?" — Ticketcontrole: je bent gekend maar heb je toegang tot dit endpoint?

Beveilig een endpoint met [Authorize]. Stuur het JWT-token mee via de header: Authorization: Bearer <token>

🚨

Kritieke volgorde in Program.cs!

UseRouting() → UseCors() → UseAuthentication() → UseAuthorization() → MapControllers()

UseAuthentication() moet ALTIJD vóór UseAuthorization() staan!

Deel 7 — React Front-end

Componenten, Props & Hooks

React bouwt websites met herbruikbare componenten. Een SPA (Single Page Application) herlaadt nooit de hele pagina — alleen de inhoud verandert.

  • Props — data van ouder naar kind, altijd read-only
  • useState — data onthouden in een component. Gebruik altijd de setter-functie, nooit direct muteren!
  • useEffect — acties na het laden. [] = eenmalig bij opstarten, [dep] = bij elke wijziging van dep
  • Controlled Component — input bestuurd door state via value={...} + onChange={...}. React is de "single source of truth".
  • Lifting State Up — verplaats state naar de gemeenschappelijke ouder als meerdere kinderen die data nodig hebben

React Router

🚨

Nooit een gewone <a>-tag!

Een gewone <a href="..."> herlaadt de volledige pagina en breekt je SPA. Gebruik altijd <Link to="..."> of <NavLink>.

  • <Link> — navigatie zonder herladen
  • <NavLink> — voegt automatisch .active CSS-klasse toe voor menu's
  • <Outlet /> — placeholder waar de child-route gerenderd wordt bij Nested Routes
  • useParams() — URL-parameters uitlezen: const { id } = useParams()
  • useNavigate() — programmatisch navigeren: navigate('/riders/' + id)

Axios, CORS & Container/Presentational Components

Axios is de bibliotheek voor API-calls vanuit React. Data zit in result.data. Maak één centrale apiClient-instantie met de baseURL.

CORS moet geconfigureerd worden in je .NET API. De browser blokkeert standaard verzoeken van een andere origin (bijv. localhost:5173 → localhost:7111).

Container Component (slim)

Haalt data op, beheert state, geeft door via props. Weet van de API.

Presentational Component (dom)

Toont alleen wat het via props krijgt. Weet niets van de API. Makkelijk herbruikbaar.
